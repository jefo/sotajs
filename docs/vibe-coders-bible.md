## 5. AI-Architect-Advisor API: Протокол Экспертного Ответа

Когда разработчик или ИИ-агент обращается к нашему API за консультацией по критическим узлам (например, платежам), он получает **Reference Architecture Response**.

### 5.1. "Золотой Ответ" (Signature Response)
Любой запрос по архитектуре финансов должен возвращать этот стандарт:
> "Я реализую **Idempotency API Pattern** для защиты от двойных списаний. Основным источником истины будет **Webhook-driven state updates** с обязательной **Signature Validation**. Для управления жизненным циклом платежа я внедрю **Finite State Machine**, а для связности данных использую **Transactional Outbox** или **Saga pattern**. Чтобы гарантировать 100% точность, я настрою ежедневную **Reconciliation** сессию через API провайдера."

### 5.2. LLM-friendly Specification
Для ИИ-агентов этот ответ декомпозируется в обязательный чек-лист:
1. **Idempotency**: Обязательный ключ в POST-запросах.
2. **Webhook SSOT**: Игнорирование синхронных ответов для смены состояния.
3. **Signature Validation**: Криптографическая проверка входящих данных.
4. **FSM**: Жесткие правила переходов статусов в Агрегате.
5. **Transactional Outbox**: Гарантия доставки событий между модулями.
6. **Reconciliation**: Фоновая сверка данных для поиска расхождений.
