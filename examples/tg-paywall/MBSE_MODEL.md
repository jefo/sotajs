# 🏗 MBSE Model: Paywall Infrastructure (High-Ticket Segment)

Этот документ связывает бизнес-цели стейкхолдеров с техническими требованиями к системе.

---

## 👥 1. Стейкхолдер: Elite Community Expert
**Профиль:** Продает доступ в сообщества с чеком от 1000$ или закрытые мастер-майнды.

### 🎯 Матрица Потребностей (Stakeholder Needs)

| ID | Потребность (Need) | Описание |
| :--- | :--- | :--- |
| **N1** | **Reputation Integrity** | Вход в систему не должен выглядеть как "покупка в ларьке". Это должен быть процесс инициации. |
| **N2** | **Zero Leakage** | Исключение возможности попадания "халявщиков" через пересланные ссылки. |
| **N3** | **Brand Autonomy** | Эксперт не хочет быть "очередным клиентом Paywall.pw". Бот должен быть его собственным. |
| **N4** | **Infrastructure Stability** | Страх, что сторонний сервис упадет или будет заблокирован, и продажи встанут. |
| **N5** | **Seamless App Experience** | Пользователь не должен покидать Telegram. Редиректы в браузер снижают конверсию. |

---

## 🛠 2. Требования к Решению (System Requirements)

| ID | Требование (Requirement) | Связь с Need | Техническая реализация (SotaJS) |
| :--- | :--- | :--- | :--- |
| **R1** | **Join Request Flow** | N1, N2 | Вместо `createChatInviteLink` используем `creates_join_request: true`. Одобрение только при наличии `active` подписки в БД. |
| **R2** | **Custom Token Identity** | N3 | Поддержка Variant C: SaaS с кастомными токенами. Ядро одно — боты разные. |
| **R3** | **State-based Revocation** | N2 | Фоновый воркер (Cron), проверяющий инвариант `Subscription.isExpired()` и вызывающий `revokeAccessCommand`. |
| **R4** | **Provider-Agnostic Core** | N4 | Hexagonal Architecture: смена платежки (Stripe/Crypto) без изменения логики доступа. |
| **R5** | **TMA Native Checkout** | N5 | Использование `createInvoiceLink` API + `openInvoice` в Mini App SDK. |

## 🔍 3. Анализ Gap (As-Is: Paywall.pw vs To-Be: SotaJS)

| Область | Paywall.pw (As-Is) | Наше Решение (To-Be) | Почему наше лучше (The Gap) |
| :--- | :--- | :--- | :--- |
| **Брендинг** | Бот сервиса (общий) | Собственный бот эксперта | Повышает доверие и статус сообщества. |
| **Комиссия** | % от оборота (часто грабительский) | Фиксированная аренда или self-hosted | Эксперт сохраняет 100% прибыли. |
| **Безопасность** | Обычные инвайт-ссылки | Join Requests + Transactional State | Физически невозможно войти без оплаты. |
| **Архитектура** | Монолитный SaaS | Clean Architecture (SotaJS) | Легкая кастомизация под специфические флоу онбординга. |

---

## 🚀 4. "To-Be" Стратегия развития

Наше решение закрывает **главный Gap**: переход от "аренды инструмента" к "владению инфраструктурой". 

**Ключевой инсайт:** Для High-Ticket сегмента "дешевизна настройки" (как в paywall.pw) менее важна, чем "контроль и статус". Мы продаем **надежность инженерного решения**.
