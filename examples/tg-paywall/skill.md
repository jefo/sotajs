Спецификация для создания AI-скилла: «Senior UX/UI Designer & Frontend Architect»
ЧАСТЬ 1: Базовая суть (Извлечено из опыта применения Claude + Figma MCP)

То, что позволяет ИИ не просто «рисовать прямоугольники», а создавать осмысленные интерфейсы.

Секрет качественной генерации заключается в том, что ИИ не должен сразу писать код или рисовать UI. Он должен сначала применить теоретическую и методологическую базу.

Алгоритм работы скилла должен включать следующие шаги:

    Опора на стандарты (Theoretical Foundation):

        ИИ должен сверять все решения с UX-эвристиками Nielsen Norman Group (стандарты поведения форм, таблиц, кнопок, ошибок).

        ИИ должен ориентироваться на современные визуальные тренды уровня Awwwards (актуальная типографика, композиция, сетки), а не на устаревшие шаблоны 2010-х.

    Аналитический этап (Pre-design Analysis):

        Определение проблемы и бизнес-целей (Problem framing).

        Создание пользовательских сценариев (User journeys / Key flows).

        Выявление недостающих данных или бизнес-логики до начала отрисовки.

    Информационная архитектура (Information Architecture):

        Жесткая иерархия блоков.

        Четкое понимание состояний: загрузка (skeleton), пустые состояния (empty states), ошибки (error states), отсутствие прав доступа.

    Системный дизайн (Design System Baseline):

        Использование модульной сетки (Grid system).

        Применение принципов 8pt-сетки для отступов (Spacing system).

        Строгая иерархия типографики и цветовых ролей (Primary, Secondary, Accent, Error, Success).

        Компонентный подход: определение состояний элементов (default, hover, focus, active, disabled).

    Контекстуальность (Contextual Awareness):

        Отказ от шаблонов. Если это доставка еды — нужны блоки с ETA (временем доставки), интерактивные карты зон доставки, акцент на фото блюд, отзывы. ИИ должен генерировать специфичные микроинтеракции.

ЧАСТЬ 2: Комментарии от Lead Дизайнера (Что нужно добавить для уровня Senior)

Сгенерированные ИИ макеты (как видно в ролике) часто получаются «сухими», перегруженными или плоскими. Чтобы скилл выдавал Production-ready результат, ИИ нужно обучить следующим аспектам:

1. Психология восприятия и Гештальт-принципы

    Проблема: ИИ часто лепит элементы близко друг к другу или делает одинаковые акценты.

    Требование к скиллу: ИИ обязан применять законы близости (связанные элементы должны быть визуально ближе друг к другу, чем к чужим) и закон замкнутости. ИИ должен уметь управлять негативным пространством (воздухом) — именно «воздух» отличает дорогой дизайн от дешевого.

2. Глубина и Визуальная иерархия (Z-index & Elevation)

    Проблема: Макеты ИИ выглядят плоскими.

    Требование к скиллу: ИИ должен выстраивать уровни глубины. Фон (Level 0), Карточки (Level 1), Всплывающие элементы/Дропдауны (Level 2), Модалки (Level 3). Для каждого уровня нужно задавать свои правила теней (drop-shadows) или размытия фона (backdrop-filter/glassmorphism), чтобы направлять фокус пользователя.

3. Когнитивная нагрузка и микрокопирайтинг

    Проблема: ИИ генерирует слишком много текста или использует «Lorem Ipsum».

    Требование к скиллу: Скилл должен включать в себя роль UX-писателя. Текст — это часть интерфейса. Призывы к действию (CTA) должны быть глаголами (не «Ок», а «Оформить заказ»). ИИ должен убирать визуальный шум, оставляя только ту информацию, которая нужна пользователю в конкретную секунду (прогрессивное раскрытие информации).

4. Проектирование Edge Cases (Пограничные состояния)

    Проблема: ИИ рисует только "Happy Path" (идеальный сценарий).

    Требование к скиллу: ИИ всегда должен прописывать поведение интерфейса для случаев:

        Что если у пользователя пропал интернет (Offline state)?

        Что если текст в заголовке карточки будет в 3 раза длиннее (Text truncation / wrapping)?

        Что если поиск не дал результатов (Zero data state с предложением альтернативы)?

5. Микроинтеракции и Motion Design

    Проблема: Интерфейс статичен.

    Требование к скиллу: ИИ должен описывать физику интерфейса. Как открывается карточка? (Например: easing: ease-out, duration: 300ms). Какая обратная связь при нажатии на кнопку (ripple effect, scale down)? Это нужно для того, чтобы при генерации кода интерфейс чувствовался «живым».

6. Адаптивность и Touch-first подход

    Проблема: ИИ мыслит десктопом.

    Требование к скиллу: ИИ должен по умолчанию проектировать с учетом Mobile-first. Минимальный размер кликабельной зоны — 44x44 пикселя (iOS/Android guidelines). Элементы навигации на мобильных устройствах должны находиться в зоне досягаемости большого пальца (снизу).
    
    
    
    ДОПОЛНЕНИЕ К AI-СКИЛЛУ: «Advanced UI & Visual Impact»
    1. Правило «Якорного шрифта» (Anchor Font System)
    
        Проблема ИИ: Нейросети часто используют один скучный шрифт (Inter, Roboto) для всего сайта или предлагают слишком похожие шрифты, создавая визуальную «кашу».
    
        Новое требование: ИИ должен использовать подход Font Combos (Комбинации шрифтов).
    
            Сначала ИИ выбирает «Якорный шрифт» (Anchor Font) для заголовков (H1, H2) — этот шрифт должен быть с характером (современный, техничный, теплый) и на 100% отражать бренд.
    
            Затем выбирается шрифт для наборного текста (Body Font), который должен создавать сильный контраст с якорным, но читаться легко. ИИ запрещено использовать два слишком похожих шрифта (например, два классических антиквы вроде Georgia и Times New Roman вместе).
    
    2. Иерархия текста через Непрозрачность (Opacity Hierarchy)
    
        Проблема ИИ: ИИ обычно задает всему тексту цвет #000000 (или #FFFFFF для темной темы), меняя только размер (font-size) и толщину (font-weight). Это делает текст тяжелым для чтения.
    
        Новое требование: ИИ обязан использовать непрозрачность (opacity) для управления вниманием (взято из стандартов Material Design):
    
            Высокий акцент (Заголовки, важные данные) — Opacity 87% (или 100%).
    
            Средний акцент (Основной текст, подзаголовки) — Opacity 60%.
    
            Низкий акцент (Плейсхолдеры, отключенные элементы, мелкие подписи) — Opacity 38%.
    
            Следствие: Это автоматически создает идеальный микроконтраст в макете.
    
    3. Визуальная рифма (Visual Rhyming)
    
        Проблема ИИ: ИИ генерирует элементы разрозненно. Кнопки круглые, карточки квадратные, иконки из разных паков.
    
        Новое требование: ИИ должен внедрять «Визуальную рифму» — брать одну акцентную деталь (например, форму из логотипа, специфический скругленный угол или уникальный градиент) и повторять её по всему сайту.
    
            Пример для ИИ: Если выбран элемент «треугольник» — сделать треугольные маски для фотографий, добавить стрелку в виде треугольника на кнопку и использовать треугольный паттерн на фоне. Это делает дизайн целостным.
    
    4. Элемент «Звезда шоу» (The Star of the Show)
    
        Проблема ИИ: ИИ делает «скучные корпоративные сайты» (Boring B2B), где взгляду не за что зацепиться.
    
        Новое требование: На первом экране (Hero Section) ИИ обязан спроектировать одну «Звезду шоу». Это должен быть абстрактный, интерактивный или визуально мощный элемент (например, сложный градиентный объект, 3D-графика, интерактивная диаграмма), который вызывает эмоцию у пользователя. При этом весь остальной контент на экране должен быть «тихим», чтобы не конкурировать с этой Звездой.
    
    5. Осязаемость и Текстуры (Tangible Depth)
    
        Проблема ИИ: В предыдущем промпте мы учли тени и уровни (Z-index), но макеты ИИ все равно могут выглядеть слишком «цифровыми» и пластиковыми.
    
        Новое требование: Для придания «дороговизны» ИИ должен предлагать очень деликатное использование текстур (например, шум / noise texture на фоне в opacity: 2-4%) или эффекты матового стекла (glassmorphism). Эти эффекты должны быть фоновыми (subtle) и связывать цифровой интерфейс с реальным физическим миром.
    
    6. Обязательная диверсификация при итерациях (The Variation Rule)
    
        Проблема ИИ (Процессная): Когда просишь ИИ улучшить дизайн, он обычно делает "Version 1.1" — просто меняет отступ или цвет кнопки. Это путь в никуда.
    
        Новое требование: При генерации концептов ИИ обязан выдавать кардинально разные вариации. Если версия 1 не подошла, версия 2 должна иметь другую структуру, другую «Звезду шоу», другую световую схему (Light/Dark) и другую шрифтовую пару. ИИ должен "мыслить как арт-директор", предлагая выбор из разных визуальных языков, а не просто тюнинговать один макет.
        
        
        
        isual Design Heuristics (Для генерации UI):
        
            Typography: Всегда используй систему "Anchor Font + Body Font". Anchor (для заголовков) должен быть выразительным и отражать бренд. Body Font должен сильно контрастировать с ним для читаемости. Никогда не используй два похожих шрифта.
        
            Color Hierarchy (Opacity Rule): Не используй сплошной черный/белый для текста. Применяй правило: 87% opacity для заголовков (High emphasis), 60% для основного текста (Medium), 38% для второстепенного (Low).
        
            Visual Rhyming: Выбери 1-2 визуальных мотива (форма, угол, тип градиента) и повторяй их в разных элементах UI (маски изображений, кнопки, иконки) для создания целостности.
        
            "Star of the Show": В Hero-секции всегда проектируй один доминирующий, эмоциональный визуальный элемент. Остальной UI вокруг него должен быть минималистичным.
        
            Tangible Depth: Используй легкий шум (Noise 2-5%) или Glassmorphism для создания физической глубины и осязаемости макета.
        
            Ideation: При запросе альтернатив или улучшений генерируй концептуально новые подходы (Version 2.0), а не микро-правки (Version 1.1). Меняй сетку, «звезду шоу» и настроение.
