# Product Requirements Document (PRD): Сервис бронирования переговорных комнат (RoomBooking MVP)

## 1. Введение и контекст

**Проблема:** В офисе возникают конфликты из-за переговорных комнат: сотрудники бронируют одно и то же время, не приходят на встречи, отменяют бронь в последний момент.

**Решение:** Простой сервис для бронирования слотов времени с политикой отмены.

---

## 2. Цели

- Устранить конфликты при бронировании (накладки).
- Обеспечить политику отмены со штрафами за позднюю отмену.

---

## 3. Пользователи

- **Сотрудник** — бронирует, отменяет, смотрит расписание.
- **Офис-менеджер** — видит статистику.

---

## 4. Функциональные требования

### 4.1. Бронирование

**Что нужно:**
- Выбор комнаты (Amber, Sapphire, Emerald).
- Выбор времени (начало и конец).
- ID пользователя.

**Правила:**
- ❗ Нельзя забронировать пересекающийся слот в той же комнате.
- ❗ Конец должен быть после начала.
- ❗ Стоимость: 500 руб/час.

**Пример:**
```
✅ 10:00-12:00 и 12:00-14:00 — OK
❌ 10:00-12:00 и 11:00-13:00 — Конфликт!
```

**Результат:**
- Успех: `{ bookingId, totalCost }`
- Ошибка: `{ error: "Time slot overlaps" }`

---

### 4.2. Отмена брони

**Что нужно:**
- Отмена по ID брони.
- Причина (текст).

**Правила:**
- ❗ Отмена ≥ 2 часов до начала — без штрафа.
- ❗ Отмена < 2 часов до начала — штраф 10%.

**Пример:**
```
Встреча в 15:00:
- Отмена в 12:30 → без штрафа
- Отмена в 13:30 → штраф 10%
```

**Результат:**
- Успех: `{ penaltyApplied, refundAmount }`
- Ошибка: `{ error: "Booking not found" }`

---

### 4.3. Расписание комнаты

**Что нужно:**
- Выбор комнаты.
- Выбор даты.

**Возвращает:**
- Список бронирований на дату.
- Занятые часы.
- Выручка.

## 7. Критерии приёмки

### AC-001: Создание брони
```
Дано: Комната свободна с 10:00 до 12:00
Когда: Бронь с 10:00 до 12:00
Тогда: Бронь создана, стоимость 1000 руб
```

### AC-002: Пересечение
```
Дано: Комната забронирована 10:00-12:00
Когда: Бронь 11:00-13:00
Тогда: Ошибка "Time slot overlaps"
```

### AC-003: Отмена без штрафа
```
Дано: Бронь на 15:00, отмена за 2.5 часа
Тогда: Штраф 0 руб
```

### AC-004: Отмена со штрафом
```
Дано: Бронь на 15:00, отмена за 1.5 часа
Тогда: Штраф 10% (100 руб)
```

### AC-005: Расписание
```
Дано: 3 брони на сегодня
Когда: Запрошено расписание
Тогда: Возвращены все 3 брони, посчитаны часы и выручка
```

---

## 8. Примеры данных

**Создание брони:**
```json
// Input
{
  "roomId": "amber",
  "userId": "user-123",
  "start": "2025-01-15T10:00:00Z",
  "end": "2025-01-15T12:00:00Z"
}

// Output
{
  "success": true,
  "bookingId": "uuid-123",
  "totalCost": 1000
}
```

**Отмена:**
```json
// Input
{
  "bookingId": "uuid-123",
  "reason": "Changed plans",
  "cancelledAt": "2025-01-15T07:00:00Z"
}

// Output
{
  "success": true,
  "penaltyApplied": 0,
  "refundAmount": 1000
}
```
